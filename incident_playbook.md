# Incident Playbook

## Phase 1: Detect
- monitors raise alert
- replay snapshot captured

## Phase 2: Contain
- freeze incentive distribution
- quarantine suspicious nodes
- restrict quorum decisions

## Phase 3: Diagnose
- run replay diff vs baseline policy
- collect evidence
- trigger dispute resolution if needed

## Phase 4: Respond
- apply targeted slashing (only with evidence)
- apply policy patch
- restore safe routing/sync policies

## Phase 5: Recover
- rebuild trust weights
- allow staged return of nodes
- release rewards gradually

## Phase 6: Publish
- mandatory postmortem
- publish replay trace hashes
